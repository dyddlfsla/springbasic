
 ◆ 스프링은 왜 만들어졌는가?

  - 핵심 개념
   스프링은 자바 언어 기반의 프레임워크,
   자바 언어의 가장 큰 특징은 객체지향 언어라는 것이다.
   스프링은 객체 지향 언어가 가진 강력한 특징을 잘 살려낼 수 있도록 도와주는 프레임워크
   즉, 좋은 객체 지향 애플리케이션을 개발할 수 있게 도와준다.



 - 핵심기술: 스프링 DI 컨테이너, AOP, 이벤트,
 - 웹 기술: 스프링 MVC, 스프링 WebFlux
 - 데이터 접근 기술: 트랜잭션, JDBC, ORM 지원, XML 지원
 - 기술 통합: 캐시, 이메일, 원격접근, 스케줄링
 - 테스트: 스프링 기반 테스트 지원
 - 언어: 자바, 코틀린, 그루비

 스프링부트
 1) 스프링을 편리하게 사용할 수 있도록 지원, 최근에는 프로젝트 개발 시 기본적으로 사용.
 2) 단독으로 실행할 수 있는 스프링 애플리케이션을 쉽게 생성
 3) Tomcat 같은 웹 서버를 내장하고 있으므로 별도의 웹 서버를 설치하지 않아도 됨
 4) 손쉬운 빌드 구성을 위한 starter 종속성 제공
 5) 스프링과 서드파티 라이브러리 자동 구성
 6) 메트릭, 상태 확인, 외부 구성 같은 프로덕션 준비 기능 제공
 7) 관례에 의한 간결한 설정
